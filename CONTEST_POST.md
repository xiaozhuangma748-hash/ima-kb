## 1. Demo 简介

**是什么**：IMA 个人知识库（ima-kb）是一个**本地优先**的个人知识管理工具，把电脑里散落的资料（PDF / Word / Excel / PPT / Markdown / 网页 / 代码 / 图片 / 扫描件）整理成一个**可搜索、可问答、可分析**的本地知识库。提供 CLI 命令、交互式 REPL、Web 后台三套入口，共享同一套数据。

**面向谁**：
- 研究人员/学生：管理论文、笔记、资料，快速检索和问答
- 开发者：管理代码片段、技术文档、项目资料
- 政企从业者：管理政策文件、汇报材料、数据报表（项目原型场景为殡葬政策知识库）

**核心功能**（3 个）：

**① 混合检索 + RAG 问答**
- 四层流水线：BM25 关键词 + 向量语义（bge-small-zh-v1.5）+ RRF 融合 + LLM 重排序
- 检索性能优化：BM25/向量并发检索 + 语义缓存（L1 精确 + L2 相似度）+ 查询路由（闲聊跳过检索）
- RAG 问答带引用溯源（`[1] [2]` 标注真实文档标题和段落号）
- 流式输出 + LaTeX 公式自动清理，多轮对话

**② 40+ 格式一键入库**
- 文档：PDF / Word / Excel / PPT / Markdown / TXT / HTML
- 代码：20+ 种语言（Python / JS / Go / Rust / Java ...）
- 图片：PaddleOCR（主）+ Tesseract（降级）双引擎，支持扫描件
- 自动去重（SHA256）+ LLM 自动标签 + 智能分块

**③ 虚拟宠物管理员 + Web 后台**
- 所有 AI 交互由 PetAdministrator 统一编排，串联检索→重排→Prompt→LLM→引用→记忆→经验
- 4 种人格风格（学者/战士/工匠/中性）+ 像素风 ASCII 艺术
- Web 后台 7 页面：AI 问答（SSE 流式）/ 入库 / 搜索 / 数据分析 / 仪表盘 / 知识图谱 / 宠物管理

## 2. Demo 创作思路

**灵感来源**：
日常工作中，电脑里散落着大量资料——PDF 政策文件、Word 汇报、Excel 数据表、PPT 材料、网页收藏、代码片段。找资料靠文件夹翻找 + 全文搜索，问答靠人工阅读，分析靠手动整理。资料越多，越难找到、越难用起来。

**想解决的问题**：
- **找不到**：系统搜索只能按文件名匹配，无法按内容语义检索
- **看不懂**：PDF 扫描件、图片中的文字无法被搜索
- **不会用**：找到资料后还要人工阅读提炼，没有 AI 辅助问答
- **不互通**：CLI 党、Web 党、终端党各有所好，单一入口无法满足

**为什么做这个方向**：
- **本地优先**：个人/政企资料涉及隐私，不能上传到云端服务。所有数据（SQLite + 向量库 + 原文件）都在本机 `storage/` 目录，断网可用
- **混合检索**：纯关键词匹配不到同义表达，纯向量漏掉精确匹配，两者融合 + LLM 重排才能兼顾召回率和准确率
- **多入口共享**：CLI 批量操作 + REPL 交互问答 + Web 可视化展示，三套入口共享同一套 core 服务，覆盖不同使用场景

### 技术亮点

**① 四层混合检索流水线 + 性能优化**
```
用户提问 → 查询路由 → 语义缓存 → BM25 关键词 + 向量语义（并发） → RRF 融合(k=60) → LLM 重排序 → Top-K 结果
```
- BM25：NFKC 文本归一化 + jieba 多模式分词 + IDF 截断 + b=0.5 降低长文档惩罚
- 向量：ChromaDB + bge-small-zh-v1.5（本地运行，HF 镜像下载）
- RRF 融合：k=60，两路结果分数归一化
- LLM 重排：对候选打 0-10 分，三级 fallback 解析 LLM 输出
- **查询路由**：闲聊/问候直接走 LLM，跳过检索（1.5-5s → ~200ms）
- **语义缓存**：L1 精确 + L2 embedding 相似度，命中率 40-60%，重复/相似问题 <10ms
- **并发检索**：BM25 与向量并行执行，粗排 top 50 再精排，降低延迟 30-50%

**② 虚拟宠物管理员编排层**
所有 AI 交互统一走 PetAdministrator：
```
检索 → 重排 → build_system_prompt(按人格风格) → LLM → 引用溯源 → 记忆更新 → 宠物经验
```
- 4 种人格：scholar（深度分析）/ warrior（直接行动）/ artisan（结构化）/ neutral
- 35 个 ASCII 艺术文件，按风格和等级切换
- 失败降级：PetAdministrator → RAGChain → 纯 LLM 对话

**③ 多层降级机制**
任何单点故障都不会让整体崩溃：
- 向量库不可用 → 纯 BM25
- 重排失败 → 用原混合结果
- LLM 失败 → 降级提示
- 宠物管理员失败 → RAGChain → 纯 LLM

**④ 输出可读性防御**
LLM 偶尔输出 LaTeX 公式（如 `$$6520.92 \\times 2 = \\mathbf{13,041.84元}$$`），三层防护保证终端可读：
- **Prompt 层**：system prompt 绝对禁止 LaTeX 语法
- **渲染层**：流式输出时实时清理 `$$`、`times`、`mathbf{}` 等符号
- **缓存层**：答案写入/命中前都执行清理

**⑤ 跨会话记忆系统**
- 每轮对话后 LLM 自动提取：用户偏好 / 话题 / 未解决问题 / 事实
- 按会话名隔离存储，去重合并
- 新会话自动注入历史记忆，实现"跨会话记忆"

## 开发心得

**TRAE 让我做到了原本做不到的事**：

1. **从 0 到 1 快速落地**：一个人 7 个阶段完成了一个包含 CLI + REPL + Web + 向量检索 + 知识图谱 + 记忆系统的完整产品，没有 TRAE 的规划能力和代码生成能力，这个工作量至少需要一个团队

2. **跨领域技术整合**：BM25 算法、向量检索、OCR、知识图谱、FastAPI、prompt_toolkit……这些跨越多个技术栈的能力，TRAE 帮我整合到了一个项目里

3. **工程化质量**：323+ 测试、多层降级、原子写入、懒加载、连接池……这些工程化细节都是在 TRAE 的协助下逐步打磨的

4. **踩过的坑**：
   - Agnes LLM 偶尔返回空 chunk（只有 usage 数据）→ 在 client.py 跳过空 choices
   - 中国大陆下载 HF 模型超时 → 设置 HF_ENDPOINT=https://hf-mirror.com
   - pip install -e . 找不到 py-modules → pyproject.toml 显式声明 py-modules
   - BM25 全角/半角字符匹配不上 → NFKC 归一化 + 多模式分词并集
   - LLM 输出 LaTeX 公式在终端显示乱码 → 加 system prompt 禁止 + 渲染层/缓存层双重清理
