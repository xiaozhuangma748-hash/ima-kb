# AI 问答模块 Redesign 设计文档

## 背景

当前 AI 问答页采用左侧主区域 + 右侧固定边栏的布局，存在以下视觉问题：
- 页面头部厚重，占用有效空间
- 聊天区过宽，长回答阅读疲劳
- 右侧「引用来源」「推荐问题」卡片样式臃肿，层级不清
- 消息气泡和 Markdown 渲染缺乏现代 AI 聊天产品的精致感

## 目标

参考 ChatGPT / Claude 等主流 AI 聊天界面，将 AI 问答页 redesign 为：
- 居中窄列、沉浸阅读的聊天布局
- 可折叠的右侧来源面板
- 更现代的输入框和消息气泡
- 更清晰的引用溯源交互

## 设计决策

### 1. 整体布局

- **隐藏 AI 问答页头**：进入页面直接是聊天界面，无标题栏
- **聊天区居中**：最大宽度 `760px`，左右自动留白
- **输入框悬浮底部**：固定在视口底部，圆角大胶囊样式，带柔和阴影
- **右侧可折叠来源面板**：默认隐藏，宽度 `320px`，从右侧滑入

### 2. 消息区域

- **AI 消息**：左侧带头像/图标，内容区域无气泡，直接平铺 Markdown
- **用户消息**：右侧圆角气泡，背景 `rgba(6, 182, 212, 0.08)`，边框 `rgba(6, 182, 212, 0.25)`
- **引用编号**：圆角药丸标签，浅青色背景，hover 变深，点击打开右侧来源面板
- **Markdown 渲染优化**：
  - 表格带边框、圆角、斑马纹
  - 列表缩进清晰
  - 标题 `h1/h2/h3` 字号层级分明
  - 代码块带背景和高亮

### 3. 来源面板（右侧抽屉）

- **触发方式**：
  - 点击答案中的 `[n]` 引用编号
  - 输入框附近的「引用来源」常驻按钮
- **面板内容**：
  - 顶部：标题 + 关闭按钮
  - 每个来源：编号徽章 + 文档标题 + 片段摘要 + 相关度
  - 点击来源项高亮对应引用编号
- **关闭方式**：点击遮罩、点击关闭按钮、再次点击来源按钮

### 4. 空状态

- 页面中央显示渐变标题「今天有什么想了解的？」
- 推荐问题做成横向 chips，位于输入框上方
- 不再占用右侧边栏

### 5. 颜色方案

- **品牌强调色**：保留橙色 `#F59E0B`，用于发送按钮、激活状态
- **用户消息**：青色 `#06B6D4` 系
- **引用标签**：浅青色背景 + 青色文字
- **AI 消息**：纯白/浅灰背景，黑色文字
- **背景**：保持 `#f7f8fa`

### 6. 动画与交互

- 消息进入：淡入 + 上滑 `0.2s`
- 来源面板：从右侧滑入 `0.25s`，带遮罩层淡入
- 流式输出：呼吸光标动画
- 发送按钮：loading 态禁用 + 旋转图标

## 涉及文件

- `web/templates/index.html`：HTML 结构调整 + CSS 样式
- `web/static/js/qa.js`：消息渲染、来源面板交互、引用高亮逻辑

## 验收标准

- [ ] 进入 AI 问答页无页头，聊天区居中
- [ ] 右侧来源面板默认隐藏，点击引用编号可滑出
- [ ] 用户消息显示为右侧气泡，AI 消息左侧带头像无气泡
- [ ] Markdown 表格、列表、标题渲染样式清晰
- [ ] 空状态推荐问题显示在输入框上方
- [ ] 点击来源面板中的卡片可高亮对应引用编号
- [ ] 响应式：面板在窄屏下全宽显示
